Database Design
The Database design is a process of developing a data model using data modeling concepts. The logical database design is a process of constructing a data model of information used on a specific data model but independent of physical considerations. The physical database design covers all required parameters to generate Data Definition Language (DDL) which will be used to create database. Entity-Relationship Diagrams (ER Diagrams)

The Entity Relationship Diagram is primarily a presentation model. This is a high-level data model that shows the major entities and relationships. Many-to-many relationship is allowed and keys are generally not included. 

The objectives of the ERD are :
  • Understand the concept of conceptual database design techniques. 
  • Understand Entity-Relationship notation. 
  • Create an Entity – Relationship diagram to conceptually represent a database.

Unified Modeling Language Diagrams (UML Diagrams) with Computer Aided Software Engineering Tools (CASE Tools)

UML stands for Unified Modeling Language. It is a modeling language in  object-oriented software engineering. It includes a set of graphic notation techniques to create visual models. UML's structure diagram includes Class diagram, Component diagram, Composite Structure diagram, Deployment diagram, Object diagram Package diagram, and Profile diagram. 

The Computer Aided Softer Engineering tools (CASE Tools)  like IBM Rational Rose, Dia, Visio, etc. can be used for UML modeling. In GIS 550, a modeling tool called Dia is used for UML diagram. The objects of UML diagram are:
  • Understand the concept of going from conceptual design to logical design. 
  • Understand the purpose and structure of data definition language (DDL). 
  • Create and populate a RDBMS database.

Cardinality Concepts

Cardinality is the relational property. It  defines number of  instances appear in a child table for each corresponding instance in the parent table. Different Cardanility descriptions are :

  • One to Zero, One, or More (Mandatory)
  • One to One or More (Mandatory)
  • One to Zero or One (Mandatory)
  • Zero or One to Zero, one or more (Optional)
  • Zero or one to zero or one. (Optional)

Relationship Properties and Referential Integrity

Relationship properties describe constrains on creating, modifying, and deleting instances. Referential integrity ensures that relationships between tables remain consistent during Insert, Update, and Delete operations.

Cardinality :

Cardinality defines number of instances appear in a child table for each corresponding instance in the parent table.

Primary Key:

The primary key uniquely identifies each instance in the entity. It constraints database table for unique identification of the record.

Foreign Key:

Foreign key in one table point to a primary key in another table. It is a referential integrity constraints between parent and child tables.

Table Properties

Primary Key (Simple or Composite)

A primary key is used to uniquely identify each row in a table. A primary key can consist of one or more columns on a table. 

 A simple primary key is made up of a single column only. This is applicable  when a single column is able to identify each record uniquely. There won't be  any null value in this column as well.

When two or more columns fields are used as a primary key, they are called a  composite key. Each single column's data can be duplicated but the combination  values of these columns cannot be duplicated.

Data Definition Language (DDL)

SQL statements are divided into two major categories: data definition language (DDL) and data manipulation language (DML). A data definition language  is a syntax for defining data structures, especially database schemas.Majot DDL statement types are Create Statement, Drop Statement, and Alter Statement.

Sample of the SQL creating gamefish table:

create table tbl_gamefish (
   unique_site_visit_id varchar(20)  not null,
   data_sheet_number    integer      not null,
   species_code         varchar(20),
   run_number           integer,
   mass_weight          numeric(10,2),
   notes                varchar(255),
   constraint pk_tbl_gamefish primary key (unique_site_visit_id,data_sheet_number)
)   ;
